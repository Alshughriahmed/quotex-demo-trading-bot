# Quotex Demo Telegram Bot

بوت Telegram لإدارة إشارات تداول وتجربة تنفيذ صفقات DEMO على Quotex.

> تنبيه: هذه نسخة تطويرية للتجربة على DEMO فقط. لا توجد أي ضمانات ربح. لا تشغل REAL قبل اختبار طويل وإدارة مخاطر صارمة.

## ما تم تنظيفه

هذه النسخة لا تحتوي على:

- `.env` الحقيقي
- `session.json`
- قواعد بيانات `*.db`
- السجلات `logs/`
- النسخ الاحتياطية `backups/`
- أي توكن أو كلمة مرور مدمجة داخل الكود

## التحسينات الأولى

- إزالة بيانات Quotex الافتراضية من `database.py`.
- إصلاح احترام الفاصل الزمني بين الصفقات في `TradingRunner.can_scan_by_schedule`.
- إضافة `.gitignore` مناسب.
- إضافة `.env.example`.
- إضافة `requirements.txt`.
- إضافة فحص أمان محلي يمنع رفع ملفات التشغيل والأسرار بالخطأ.

## التشغيل المحلي

من داخل مجلد المشروع:

```bash
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# Windows PowerShell: .venv\Scripts\Activate.ps1
pip install -r requirements.txt
cp .env.example bot/.env
```

املأ `bot/.env`:

```env
TELEGRAM_BOT_TOKEN=ضع_توكن_البوت_هنا
ADMIN_TELEGRAM_IDS=ضع_user_id_الخاص_بك
SIGNALS_CHAT_ID=ضع_chat_id_للمجموعة_اختياري
DATABASE_PATH=data.db
```

ثم شغل:

```bash
cd bot
python main.py --init-db
python main.py --check-config
python main.py --check
python main.py
```

## إعداد مجموعة الإشارات في Telegram

يمكن ضبط مجموعة الإشارات بطريقتين:

1. من ملف `bot/.env` عبر `SIGNALS_CHAT_ID`.
2. من لوحة التحكم داخل Telegram عبر: `👥 الإدارة` ثم `➕ إضافة مجموعة`.

الطريقة المفضلة أثناء التشغيل هي لوحة التحكم، لأن المجموعة تُحفظ في قاعدة البيانات ويمكن تعديلها بدون تغيير ملف `.env`.

للحصول على `chat_id`:

1. أضف البوت إلى المجموعة.
2. اكتب داخل المجموعة الأمر `/id`.
3. انسخ قيمة `chat_id` التي يرد بها البوت.
4. افتح لوحة التحكم من `/menu`.
5. اختر `👥 الإدارة` ثم `➕ إضافة مجموعة`.
6. أرسل رقم `chat_id` كما هو، وغالبًا يبدأ رقم المجموعة بـ `-100`.

لعرض المجموعات التي سيستخدمها البوت:

```text
/signal_chats
```

ويمكن استخدام الاختصار:

```text
/groups
```

إذا لم تكن هناك مجموعات محفوظة في قاعدة البيانات، سيعرض الأمر قيمة `SIGNALS_CHAT_ID` من `bot/.env` إذا كانت مضبوطة.

بعد ذلك جرّب الإرسال:

```text
/test_signal
```

إذا كان لديك أكثر من مجموعة إشارات، سيحاول البوت الإرسال لكل المجموعات. إذا نجح الإرسال لبعض المجموعات وفشل في أخرى، سيعرض لك ملخصًا بعدد المجموعات الناجحة والفاشلة مع سبب الفشل لكل مجموعة. هذا يساعدك على معرفة هل المشكلة من `chat_id` قديم، أو من أن البوت غير موجود داخل مجموعة معيّنة، أو من صلاحيات Telegram.

## فحص الأمان قبل الرفع

قبل أي `git add` أو `git push` شغّل من مجلد المشروع الرئيسي:

```bash
python tools/safety_check.py
```

إذا ظهر خطأ، لا ترفع الكود قبل إصلاحه. هذا الفحص يبحث عن:

- ملفات `.env` الحقيقية.
- قواعد البيانات `*.db`.
- ملفات الجلسات مثل `session.json`.
- مجلدات التشغيل مثل `logs/` و `backups/`.
- أنماط واضحة لتوكنات أو كلمات مرور مكتوبة داخل الملفات.

## أوامر Telegram

- `/id` لمعرفة `chat_id` و `user_id`.
- `/start` أو `/menu` لفتح لوحة التحكم.
- `/status` لعرض الحالة.
- `/signal_chats` أو `/groups` لعرض مجموعات الإشارات التي سيستخدمها البوت.
- `/test_signal` لتجربة إرسال رسالة للمجموعة أو المجموعات المحفوظة.

## ملاحظات أمان

- لا ترفع `bot/.env` إلى GitHub.
- لا ترفع `data.db`.
- لا ترفع `session.json` أو `.quotex/`.
- استخدم حساب DEMO أولًا.
